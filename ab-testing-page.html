A/B Testing Interview Preparation

Key Knowledge Points

  • Fundamentals: Definition of A/B testing, its purpose (causality), and when to use it.
  • Hypothesis Formulation: Null Hypothesis ($H_0$) and Alternative Hypothesis ($H_1$).
  • Experiment Design:
    • Control vs. Treatment Groups: Randomization, ensuring groups are comparable.
    • Sample Size Calculation: Factors affecting sample size (significance level, power, minimum detectable effect, baseline conversion rate).
    • Experiment Duration: How long to run the test, considering novelty effects, seasonality, and sufficient sample size.
    • Unit of Diversion: What entity is randomized (user, session, cookie, etc.) and its implications.
  • Metric Selection:
    • Overall Evaluation Criterion (OEC) / North Star Metric: The primary metric to optimize.
    • Guardrail Metrics: Metrics to monitor for negative side effects (e.g., latency, churn, engagement on other features).
    • Proxy Metrics: When direct measurement is difficult.
  • Statistical Concepts:
    • Statistical Significance: p-value, confidence intervals, alpha ($\alpha$) level.
    • Power: Probability of correctly rejecting a false null hypothesis.
    • Minimum Detectable Effect (MDE): The smallest effect size you want to be able to detect.
  • Interpretation of Results: Drawing conclusions, understanding statistical vs. practical significance.
  • Common Pitfalls: Peeking, novelty effect, selection bias, SUTVA violation (Stable Unit Treatment Value Assumption), multiple comparisons problem.

Preparation Details

  • Practice Design Scenarios: Be ready to design A/B tests for various product changes.
  • Statistical Intuition: Develop a strong intuition for statistical concepts, not just memorizing formulas.
  • Discuss Trade-offs: Be able to articulate the trade-offs involved in A/B test design and analysis.
  • Explain Challenges: Discuss potential challenges and how to mitigate them.

Examples

Example 1: Designing an A/B Test for a New Feature

Question: You want to test a new "dark mode" feature for a mobile app. How would you design an A/B test for it?

Answer Points:

  • Hypothesis: Dark mode will increase user engagement and retention.
  • Control Group (A): Users see the current light mode.
  • Treatment Group (B): Users see the new dark mode.
  • Unit of Diversion: User ID (to ensure consistent experience for a user).
  • Primary Metrics (OEC): Average session duration, daily active users (DAU), weekly active users (WAU).
  • Guardrail Metrics: App crashes, load times, user complaints (qualitative), uninstalls.
  • Sample Size: Calculate based on desired statistical power (e.g., 80%), significance level (e.g., 0.05), and expected MDE for primary metrics.
  • Duration: Run for at least 1-2 full weeks to capture weekly usage patterns and mitigate novelty effects. Longer if MDE is small or traffic is low.
  • Randomization: Ensure users are randomly assigned to groups to minimize bias.

Example 2: Interpreting a Negative A/B Test Result

Question: Your A/B test for a new checkout flow shows a statistically significant *decrease* in conversion rate. What would you do next?

Answer Points:

  • Don't Panic: A negative result is still a result. It prevents a bad change from being rolled out.
  • Verify Data Integrity: Double-check randomization, data collection, and metric calculation for any errors.
  • Segment Analysis: Analyze the impact across different user segments (new vs. returning, device type, geography) to identify if the negative impact is localized.
  • Funnel Analysis: Break down the checkout flow into smaller steps to pinpoint where users are dropping off in the new experience.
  • Qualitative Feedback: Look at user feedback, surveys, heatmaps, or session recordings for the treatment group to understand *why* users are struggling.
  • Hypothesize & Iterate: Formulate new hypotheses based on insights, make adjustments to the feature, and run new A/B tests.
  • Rollback: If the negative impact is significant, immediately revert to the old checkout flow.

Example 3: Multiple Comparisons Problem

Question: You're running an A/B test and monitoring 20 different metrics. You find one metric that is statistically significant at $\alpha = 0.05$. What's the potential issue?

Answer Points:

  • This is the Multiple Comparisons Problem.
  • Explanation: When you perform multiple hypothesis tests (checking 20 metrics), the probability of observing at least one false positive (Type I error) increases with each additional test, even if none of the null hypotheses are truly false. For $\alpha = 0.05$, there's a 5% chance of a false positive for *each* test. With 20 tests, the family-wise error rate is much higher than 5%.
  • Mitigation:
    • Bonferroni Correction: Adjust the alpha level ($ \alpha_{new} = \alpha / N $ where N is the number of tests). This is conservative.
    • False Discovery Rate (FDR) Control: Methods like Benjamini-Hochberg are less conservative and control the expected proportion of false positives among rejected null hypotheses.
    • Pre-define Primary Metrics: Clearly define a single or a small set of primary metrics before the experiment. Other metrics are secondary and used for deeper analysis, but the decision to launch is based on primaries.
