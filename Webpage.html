数据岗位面试流程图总结

Amazon (亚马逊)

Data Scientist (DS) / Applied Scientist (AS) 面试流程

1

**第一轮：SQL 编码**

掌握：聚合、过滤、窗口函数 (row_number(), lag(), lead())、CTE (Common Table Expressions)。

特点：难度 Easy 到 Medium，常要求优化 SQL 性能。

2

**第二轮：Python/编程**

掌握：Pandas 数据处理、数据聚合和操作，可能涉及算法题 (Two Pointers, Sliding Window, BFS)。

3

**第三轮：ML/Stats (机器学习/统计)**

掌握：基础概念 (GD/SGD, Batch Size, 监督/无监督学习, 线性/逻辑回归, 损失函数, 正则化, 过拟合)、模型 (决策树, 随机森林, Boosting, 神经网络, Transformer)、因果推断 (PSM, DID, IV)、A/B Testing (设计, 指标选择, 结果分析)。

4

**第四轮：Case Study (案例分析)**

掌握：房价估值模型、电商转化率、用户行为分析系统设计、LLM 应用设计 (如客服聊天机器人)。

5

**第五轮：Behavioral Questions (BQ) / Leadership Principles (LP)**

掌握：亚马逊 LP (Customer Obsession, Ownership, Bias for Action, Dive Deep, Invent and Simplify, Are Right, A Lot, Learn and Be Curious, Deliver Results, Have Backbone; Disagree and Commit, Earn Trust 等)，要求用 STAR 原则讲述具体故事。

**面试特点:** SQL 是重中之重，通常决定技术轮能否通过；LP 贯穿整个面试过程，需充分准备多个故事；部分岗位要求系统设计。

Business Intelligence Engineer (BIE) 面试流程

1

**第一轮：SQL**

掌握：大量复杂 SQL (窗口函数, CTE, GROUP BY, CASE WHEN, JOIN)，常考滚动平均、留存率、Top N 问题。

2

**第二轮：Data Visualization (数据可视化)**

掌握：如何用 Tableau/QuickSight 展示 KPI、Dashboard 设计原则、图表选择、改进现有图表。

3

**第三轮：Business Case**

掌握：业务场景分析，如电商网站转化率提升、用户留存率可视化。

4

**第四轮：Leadership Principles (LP)**

掌握：同 DS/AS 岗位，LP 同样非常重要。

5

**第五轮：ETL/Database**

掌握：ETL 流程、数据更新频率、数据库设计（星型/雪花模型）等。

**面试特点:** SQL 难度较高，且要求优化性能；强调数据建模和可视化能力；LP 准备是关键。

Data Engineer (DE) 面试流程

1

**第一轮：SQL**

掌握：复杂 SQL 查询 (递归, 窗口函数, Join)。

2

**第二轮：Python/编程**

掌握：数据处理、算法题。

3

**第三轮：System Design**

掌握：设计数据管道 (pipeline)，如物流实时流。

4

**第四轮：Behavioral Questions (BQ)**

掌握：行为问题。

**面试特点:** SQL 和 Python 编码是核心；系统设计能力很重要。

Data Analyst (DA) / Business Analyst (BA) 面试流程

1

**第一轮：SQL**

掌握：简单的聚合、过滤、Join、Window Function。

2

**第二轮：Python**

掌握：数据操作和处理。

3

**第三轮：Metrics**

掌握：如何定义和使用指标来改变现有流程。

4

**第四轮：Behavioral Questions (BQ)**

掌握：行为问题。

**面试特点:** SQL 难度相对较低，但要求熟练；更侧重业务理解和沟通能力。

Apple (苹果)

Data Scientist (DS) / ML Data Scientist (MLDS) 面试流程

1

**第一轮：Python/编程**

掌握：LeetCode 风格算法题 (Easy 到 Medium)、Pandas 数据操作、字符串处理、函数编写 (均值、中位数、众数)。

2

**第二轮：SQL**

掌握：简单的 Join、Window Function、聚合查询。

3

**第三轮：ML/Stats (机器学习/统计)**

掌握：基础概念 (CNN, 随机森林, 模型评估指标, A/B Testing)、模型选择、LLM (Transformer 原理, 微调)。

4

**第四轮：Case Study (案例分析)**

掌握：评估模型、指标选择、产品评估。

5

**第五轮：Behavioral Questions (BQ)**

掌握：冲突处理、优先级管理、独立性、与难缠客户沟通。

**面试特点:** 面试轮次较多 (可能长达 7 轮)；招聘流程可能较长，HR 反馈不及时；MLDS 岗位深入考察机器学习和 LLM。

Data Analyst (DA) 面试流程

1

**第一轮：SQL**

掌握：基础 SQL 查询。

2

**第二轮：Python**

掌握：数据处理。

3

**第三轮：Visualization**

掌握：数据可视化。

4

**第四轮：Business Sense**

掌握：产品分析、指标设计。

**面试特点:** 更侧重数据分析和业务理解。

Software/Data Engineer (DE) 面试流程

1

**第一轮：Coding**

掌握：Python 编码 (偏向数据工程实际问题)。

2

**第二轮：Competency and Technical Discussion**

掌握：团队、角色、项目讨论。

**面试特点:** 可能不直接考 LeetCode 算法题，更侧重实际工程能力。

Google (谷歌)

Data Scientist (DS) / Product Data Scientist (PDS) / Data Scientist Research (DSR) 面试流程

1

**第一轮：SQL**

掌握：聚合、Join、窗口函数、MoM (Month-over-Month) Change、AB Test 结果分析。

2

**第二轮：Python/R 编程**

掌握：数据操作 (Pandas/dplyr)、算法题 (Anagram, Duplicate Removal)、模拟研究。

3

**第三轮：ML/Stats (机器学习/统计)**

掌握：基础概念 (Bias-Variance Tradeoff, P-value, 置信区间, 假设检验 (Chi-square test))、模型 (线性回归, 逻辑回归, 分类器评估指标 (Precision/Recall Curve), RMSE)、因果推断。

4

**第四轮：Case Study (案例分析)**

掌握：产品问题分析 (Google Meet/YouTube 用户下降)、AB Testing 设计 (Google Map 新功能)、用户增长策略、自动回复功能改进。

5

**第五轮：Data Intuition / Applied Analysis**

掌握：数据分析和直觉，如如何评估海量长尾搜索数据、如何识别类似客户。

6

**第六轮：Googleyness and Leadership (BQ)**

掌握：决策失误、团队合作、时间管理、面对分歧。

**面试特点:** 面试内容全面，涵盖 SQL、Python、ML/Stats、Case Study 和 BQ；Case Study 偏向产品，需深入思考业务逻辑；对统计学基础要求很高，有时要求手推公式。

Business Data Scientist (BDS) 面试流程

1

**第一轮：SQL**

掌握：基础 SQL 查询。

2

**第二轮：Python**

掌握：基础 Python 编程。

3

**第三轮：Stats + ML**

掌握：统计和机器学习基础。

4

**第四轮：Marketing/Ads**

掌握：可能涉及营销和广告相关的问题。

**面试特点:** 可能更侧重业务和营销数据分析。

Meta (Facebook)

Data Scientist (DS) / Data Science Analyst (DSA) / Product Data Analyst (PDA) 面试流程

1

**第一轮：SQL**

掌握：高频考题 (`content_views`, `violating_content` 表格相关的查询、连续月份购买用户、广告收入 YoY (Year-over-Year) 增长率、Oculus 相关题目)。

2

**第二轮：Python**

掌握：奇数组成最小正数、字典中查找最频繁评论、连续两年最大班级人数、书架放书问题。

3

**第三轮：Analytical Reasoning (AR)**

掌握：广告推荐模型评估、A/B Testing (CTR 提升、指标权衡、图表问题)、假账户识别 (贝叶斯计算)。

4

**第四轮：Analytical Execution (AE)**

掌握：坏账户识别、贝叶斯概率计算、LLM 模型评估指标。

5

**第五轮：Product Sense (PS)**

掌握：如何衡量平台有害内容严重性、Group Call 功能需求分析、新功能上线评估、指标选择与权衡 (如 Time Spent vs. Retention Rate)。

6

**第六轮：Behavioral Questions (BQ)**

掌握：项目转向、建设性反馈、说服他人、处理同事不适、AR (Are Right, A Lot)。

**面试特点:** 面试流程结构化，SQL 和 Product Sense 是高频考点，且问题深入；强调 A/B Testing 的设计和分析；BQ 非常注重协作和沟通；面试官专业且效率高，但有时会比较 Intensive 和 Pushy。

Data Engineer (DE) 面试流程

1

**第一轮：SQL**

掌握：书店相关题目 (销量、邀请人消费、作者统计)、连续两个月购买书籍的独立客户数量。

2

**第二轮：Python**

掌握：奇数组成最小正数、最频繁评论、连续两年最大班级人数、书架放书问题、Doordash 外卖应用相关问题。

3

**第三轮：Product Sense**

掌握：如何定义指标、如何进行可视化。

4

**第四轮：Data Modeling**

掌握：数据表结构设计、ETL 流程。

**面试特点:** SQL 和 Python 编码是核心，题目通常是套题形式；Product Sense 和 Data Modeling 也会涉及。

Microsoft (微软)

Data Scientist (DS) / Applied Scientist (AS) 面试流程

1

**第一轮：编程 (Coding)**

掌握：Python (字符串处理、数组操作、Two Pointers, Sliding Window)、SQL (聚合、Join、窗口函数)。难度 Easy 到 Medium。

2

**第二轮：ML/Stats (机器学习/统计)**

掌握：基础概念 (ML Pipeline, DL 解释, 损失函数, 计算复杂度, 回归, Attention 机制, Bias-Variance Tradeoff, 过拟合/欠拟合)、模型 (CNN vs RNN, LSTM, K-means)、A/B Testing (影响因素, 探索与利用)、概率。

3

**第三轮：Product Sense**

掌握：设计 ML 模型预测 CTR、改进模型 (维度降低、特征选择、核方法、集成学习、个性化建模、数据增强)。

4

**第四轮：Behavioral Questions (BQ)**

掌握：最具挑战的项目、团队冲突、职业目标障碍、Why Microsoft。

**面试特点:** 编码题量相对较少，但考察概念理解和优化；ML/Stats 考察范围广；BQ 同样重要，深入挖掘故事。
