SQL Interview Preparation

Key Knowledge Points

  • Basic to Advanced SQL Queries: SELECT, FROM, WHERE, GROUP BY, HAVING, ORDER BY.
  • Joins: INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN, CROSS JOIN, Self-Join. Understand when to use each.
  • Window Functions: ROW_NUMBER(), RANK(), DENSE_RANK(), NTILE(), LAG(), LEAD(), SUM() OVER(), AVG() OVER(). Essential for complex analytical queries.
  • Subqueries and Common Table Expressions (CTEs): Understand their use for breaking down complex queries and improving readability.
  • Aggregations: COUNT(), SUM(), AVG(), MIN(), MAX().
  • Data Types and Constraints: Understanding common data types (INT, VARCHAR, DATE, BOOLEAN) and constraints (PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL).
  • Indexing: Basic understanding of how indexes work and their impact on query performance.
  • SQL Performance Optimization: How to identify bottlenecks and optimize slow queries (e.g., using EXPLAIN PLAN).

Preparation Details

  • Practice Platforms: Utilize platforms like LeetCode (SQL section), HackerRank, or DataLemur for daily practice. Focus on problems ranging from Easy to Medium-Hard.
  • Real-world Scenarios: Think about how SQL is used in business contexts (e.g., calculating user retention, sales trends, funnel analysis).
  • Explain Your Logic: Be prepared to explain your thought process and query logic step-by-step during the interview.
  • Edge Cases: Consider edge cases for your queries (e.g., empty tables, NULL values, duplicate entries).

Examples

Example 1: Find the Nth Highest Salary

Problem: Write a SQL query to find the Nth highest salary from an 'Employees' table.

-- Assume N = 2 (find the 2nd highest salary)
SELECT DISTINCT Salary
FROM Employees
ORDER BY Salary DESC
LIMIT 1 OFFSET 1;

-- Using Window Function (more robust for Nth highest)
WITH RankedSalaries AS (
    SELECT
        Salary,
        DENSE_RANK() OVER (ORDER BY Salary DESC) as rnk
    FROM
        Employees
)
SELECT Salary
FROM RankedSalaries
WHERE rnk = 2;

Example 2: Monthly Active Users (MAU)

Problem: Calculate the number of unique monthly active users for each month from a 'UserActivity' table with 'user_id' and 'activity_date'.

SELECT
    strftime('%Y-%m', activity_date) AS activity_month,
    COUNT(DISTINCT user_id) AS monthly_active_users
FROM
    UserActivity
GROUP BY
    activity_month
ORDER BY
    activity_month;

Example 3: Cohort Retention Rate

Problem: Calculate the 1-month retention rate for users based on their signup month.

-- Assuming 'users' table with 'user_id', 'signup_date'
-- and 'user_activity' table with 'user_id', 'activity_date'

WITH UserSignupMonth AS (
    SELECT
        user_id,
        strftime('%Y-%m', signup_date) AS cohort_month
    FROM
        users
),
MonthlyActivity AS (
    SELECT DISTINCT
        user_id,
        strftime('%Y-%m', activity_date) AS activity_month
    FROM
        user_activity
)
SELECT
    usm.cohort_month,
    COUNT(DISTINCT usm.user_id) AS total_cohort_users,
    COUNT(DISTINCT CASE WHEN ma.activity_month = strftime('%Y-%m', date(usm.cohort_month || '-01', '+1 month')) THEN usm.user_id END) AS retained_users_month1,
    CAST(COUNT(DISTINCT CASE WHEN ma.activity_month = strftime('%Y-%m', date(usm.cohort_month || '-01', '+1 month')) THEN usm.user_id END) AS REAL) * 100 / COUNT(DISTINCT usm.user_id) AS retention_rate_month1
FROM
    UserSignupMonth usm
LEFT JOIN
    MonthlyActivity ma ON usm.user_id = ma.user_id
GROUP BY
    usm.cohort_month
ORDER BY
    usm.cohort_month;
